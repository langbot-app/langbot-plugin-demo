# ScheNotify

กำหนดเวลาการแจ้งเตือนด้วยภาษาธรรมชาติ

## คุณสมบัติ

ScheNotify คือปลั๊กอิน LangBot ที่ช่วยให้ผู้ใช้สามารถตั้งค่าการแจ้งเตือนตามเวลาที่กำหนดผ่านการโต้ตอบด้วยภาษาธรรมชาติกับ LLM

### คุณสมบัติหลัก

- **การโต้ตอบด้วยภาษาธรรมชาติ**: ทำความเข้าใจความตั้งใจในการกำหนดเวลาของผู้ใช้ผ่าน LLM
- **การวิเคราะห์เวลาอัจฉริยะ**: รับเวลาปัจจุบันและคำนวณเวลาการแจ้งเตือนโดยอัตโนมัติ
- **รองรับหลายภาษา**: รองรับข้อความแจ้งเตือนทั้งภาษาจีนและภาษาอังกฤษ
- **คำสั่งจัดการกำหนดการ**: ดูและลบการแจ้งเตือนที่กำหนดไว้
- **การแจ้งเตือนอัตโนมัติ**: ส่งข้อความแจ้งเตือนโดยอัตโนมัติเมื่อถึงเวลาที่กำหนด

## การกำหนดค่า

### การตั้งค่าภาษา

คุณสามารถเลือกภาษาสำหรับข้อความแจ้งเตือนในการกำหนดค่าปลั๊กอิน:

- `zh_Hans` (จีนตัวย่อ) - ค่าเริ่มต้น
- `en_US` (อังกฤษ)

## การใช้งาน

### 1. กำหนดเวลาผ่าน LLM

เพียงบอกกำหนดการของคุณกับ LLM ด้วยภาษาธรรมชาติ:

**ตัวอย่าง:**
```
เตือนฉันให้เข้าประชุมพรุ่งนี้บ่าย 3 โมง
เตือนฉันให้ส่งรายงานเวลา 9 โมงเช้าวันมะรืน
เตือนฉันให้ทานมื้อเที่ยงตอนเที่ยงวันจันทร์หน้า
เตือนฉันเกี่ยวกับอาหารค่ำวันคริสต์มาสเวลา 2024-12-25 18:00
```

LLM จะดำเนินการโดยอัตโนมัติ:
1. เรียกใช้ `get_current_time_str` เพื่อรับเวลาปัจจุบัน
2. วิเคราะห์สำนวนเวลาของคุณและแปลงเป็นรูปแบบมาตรฐาน
3. เรียกใช้ `schedule_notify` เพื่อสร้างการแจ้งเตือน

### 2. ดูการแจ้งเตือนที่กำหนดไว้

ใช้คำสั่งเพื่อดูการแจ้งเตือนที่กำหนดไว้ทั้งหมด:

```
!sche
```

ตัวอย่างผลลัพธ์:
```
[Notify] การแจ้งเตือนที่กำหนดไว้:
#1 2024-12-25 18:00:00: อาหารค่ำวันคริสต์มาส
#2 2024-12-26 09:00:00: ส่งรายงาน
```

### 3. ลบการแจ้งเตือน

ใช้คำสั่งเพื่อลบการแจ้งเตือนที่ระบุ (โดยใช้หมายเลขจาก `!sche`):

```
!dsche i <หมายเลข>
```

ตัวอย่าง:
```
!dsche i 1   # ลบการแจ้งเตือนที่ 1
```

## ส่วนประกอบ

### เครื่องมือ

1. **get_current_time_str** - รับเวลาปัจจุบัน
   - รูปแบบการส่งคืน: `YYYY-MM-DD HH:MM:SS`
   - LLM ต้องเรียกใช้เครื่องมือนี้ก่อนตั้งค่าการแจ้งเตือน

2. **schedule_notify** - กำหนดเวลาการแจ้งเตือน
   - พารามิเตอร์: สตริงเวลา, ข้อความแจ้งเตือน
   - รับข้อมูลเซสชันโดยอัตโนมัติจากพารามิเตอร์ session ของเครื่องมือ

### คำสั่ง

1. **sche** (นามแฝง: s) - แสดงรายการการแจ้งเตือนที่กำหนดไว้ทั้งหมด
2. **dsche** (นามแฝง: d) - ลบการแจ้งเตือนที่ระบุ

## รายละเอียดทางเทคนิค

- ช่วงเวลาการตรวจสอบ: ทุก 60 วินาที
- ความแม่นยำของเวลา: ระดับนาที (ตรวจสอบทุกนาที)
- ข้อมูลเซสชัน: รับโดยอัตโนมัติผ่านพารามิเตอร์ session ของเครื่องมือ
- การจัดเก็บข้อมูล: ปัจจุบันใช้การจัดเก็บในหน่วยความจำ (ข้อมูลจะหายไปเมื่อเริ่มต้นใหม่)

## ตัวอย่างการสนทนา

**ผู้ใช้:** เตือนฉันให้เข้าประชุมพรุ่งนี้บ่าย 2 โมง

**LLM:** ได้เลยครับ ผมจะตั้งค่าการแจ้งเตือนให้คุณ

*[LLM เรียกใช้ get_current_time_str]*
*[LLM เรียกใช้ schedule_notify(time_str="2024-12-26 14:00:00", message="เข้าประชุม")]*

**LLM:** เรียบร้อยครับ! ผมจะเตือนคุณเมื่อถึงเวลา 2024-12-26 14:00:00: เข้าประชุม

*[ในวันถัดไปเวลาบ่าย 2 โมง]*

**บอท:** [Notify] เข้าประชุม

## หมายเหตุ

- เวลาการแจ้งเตือนต้องเป็นเวลาในอนาคต เวลาในอดีตจะถูกปฏิเสธ
- ข้อความแจ้งเตือนจะถูกส่งไปยังเซสชันเดิมที่มีการตั้งค่าการแจ้งเตือน
- การแจ้งเตือนที่ยังไม่ได้ส่งจะหายไปหลังจากเริ่มปลั๊กอินใหม่ (จะรองรับการจัดเก็บข้อมูลถาวรในเวอร์ชันอนาคต)
