# ScheNotify

使用自然語言排程通知

## 功能特性

ScheNotify 是一款 LangBot 插件，允許用戶通過與 LLM 的自然語言交互來設置定時提醒。

### 主要功能

- **自然語言交互**：通過 LLM 理解用戶的排程意圖
- **智能時間解析**：自動獲取當前時間並計算提醒時間
- **多語言支持**：支持中文和英文提醒消息
- **排程管理指令**：查看和刪除已排程的提醒
- **自動通知**：在預定時間自動發送提醒消息

## 配置

### 語言設置

您可以在插件配置中選擇提醒消息的語言：

- `zh_Hans` (簡體中文) - 預設
- `en_US` (英文)

## 使用方法

### 1. 通過 LLM 排程

只需用自然語言告訴 LLM 您的行程：

**範例：**
```
提醒我明天下午 3 點開會
提醒我後天早上 9 點提交報告
提醒我下週一中午 12 點吃午飯
提醒我 2024-12-25 18:00 聖誕晚餐
```

LLM 將自動：
1. 調用 `get_current_time_str` 獲取當前時間
2. 解析您的時間表達式並轉換為標準格式
3. 調用 `schedule_notify` 創建提醒

### 2. 查看已排程的提醒

使用指令查看所有已排程的提醒：

```
!sche
```

範例輸出：
```
[Notify] 已排程的提醒：
#1 2024-12-25 18:00:00: 聖誕晚餐
#2 2024-12-26 09:00:00: 提交報告
```

### 3. 刪除提醒

使用指令刪除特定提醒（使用來自 `!sche` 的編號）：

```
!dsche i <編號>
```

範例：
```
!dsche i 1   # 刪除第 1 個提醒
```

## 組件

### 工具 (Tools)

1. **get_current_time_str** - 獲取當前時間
   - 返回格式：`YYYY-MM-DD HH:MM:SS`
   - LLM 在設置提醒前必須調用此工具

2. **schedule_notify** - 排程通知
   - 參數：時間字符串、提醒消息
   - 從工具的 session 參數自動獲取會話資訊

### 指令 (Commands)

1. **sche** (別名: s) - 列出所有已排程的提醒
2. **dsche** (別名: d) - 刪除指定的提醒

## 技術細節

- 檢查間隔：每 60 秒一次
- 時間精度：分鐘級（每分鐘檢查一次）
- 會話資訊：通過工具的 session 參數自動獲取
- 持久化：目前使用內存存儲（重啟後丟失）

## 範例對話

**用戶：** 提醒我明天下午 2 點參加會議

**LLM：** 好的，我為您設置一個提醒。

*[LLM 調用 get_current_time_str]*
*[LLM 調用 schedule_notify(time_str="2024-12-26 14:00:00", message="參加會議")]*

**LLM：** 完成！我將在 2024-12-26 14:00:00 提醒您：參加會議

*[次日下午 2 點]*

**機器人：** [Notify] 參加會議

## 注意事項

- 提醒時間必須在未來，過去的時間將被拒絕
- 提醒消息將發送到設置提醒時所在的同一個會話
- 插件重啟後未發送的提醒將丟失（未來版本將支持持久化）
