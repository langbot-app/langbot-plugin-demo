# GroupChatSummary

使用 LLM 摘要群組聊天訊息。不再錯過重要討論。

## 功能

- **訊息收集**：自動記錄所有群組訊息
- **手動摘要**：使用 `!summary` 指令獲取即時摘要
- **基於時間的摘要**：摘要最近 N 小時內的訊息
- **LLM 工具**：當使用者詢問「我錯過了什麼？」時，AI 可以調用摘要工具
- **自動摘要**：可選在累積 N 條訊息後觸發摘要
- **持久化儲存**：訊息歷史在插件重啟後仍然存在

## 指令

| 指令 | 描述 |
|---------|-------------|
| `!summary [count]` | 摘要最近 N 條訊息 (預設：100) |
| `!summary hours <N>` | 摘要最近 N 小時內的訊息 |
| `!summary status` | 顯示訊息緩衝區狀態 |
| `!summary clear` | 清除儲存的訊息 |

## 配置

| 選項 | 預設值 | 描述 |
|--------|---------|-------------|
| Max Messages | 500 | 每個群組儲存的最大訊息數 |
| Default Summary Count | 100 | 預設摘要的訊息數 |
| Auto Summary | Off | 每 N 條訊息自動摘要 |
| Auto Summary Threshold | 200 | 自動觸發前的訊息數 |
| Language | Chinese | 摘要輸出語言 |

## 工作原理

1. 插件監聽所有群組訊息並將其儲存在記憶體中（持久化到儲存）
2. 當被觸發（指令、工具調用或自動）時，它會格式化訊息並將其發送到您配置的 LLM
3. LLM 生成包含關鍵主題、決策和行動項目的結構化摘要

## 範例
