# 1日の会話制限プラグイン

LangBot で**セッションごと**の1日の会話回数を制限し、制限の確認・設定・リセットができる
組み込みの管理**ページ**を提供します。本番環境対応。

## 機能

- **セッションごとの1日制限** — 各個人チャット／グループを個別にカウント
- **管理ページ**（LangBot 管理画面内）で以下が可能：
  - 追跡中の全セッションを本日の使用量とプログレスバー付きで表示
  - **新規セッションの既定上限**を設定
  - **セッションごとに上限を上書き**
  - 1つのセッションのカウンターを**手動リセット**、または一括リセット
  - セッションの追跡を停止（削除）
  - 上限メッセージ、サイレントモード、リセットのタイムゾーン・時刻を編集（再起動不要）
- **サイレントモード** — 上限超過メッセージを返信せず破棄
- **タイムゾーン対応リセット** — 日次ロールオーバーの UTC オフセットと時刻を設定可能
- **永続化** — すべての状態はプラグインストレージに保存され、再起動後も保持

## コンポーネント

| パス | 種類 | 役割 |
|------|------|------|
| `main.py` | Plugin | 共有状態、カウント／制限／リセットのロジック、永続化 |
| `components/event_listener/` | EventListener | 各会話をカウントし、上限到達時にブロック |
| `components/pages/manage/` | Page | 管理 UI（セッション、既定値、手動リセット） |

## 設定

以下は**初期既定値**です。すべて管理ページからリアルタイムで編集できます。

| オプション | 型 | 既定値 | 説明 |
|------|------|--------|------|
| `daily_limit` | integer | `50` | セッションごとの1日の既定上限。`0` = 無制限。 |
| `limit_message` | string | `您今天的对话次数已达上限，请明天再来吧~` | 上限超過時に表示されるメッセージ。 |
| `reset_timezone_offset` | integer | `8` | 日次リセットの UTC オフセット（時間）。 |
| `reset_hour` | integer | `0` | カウンターがリセットされる時刻（設定タイムゾーン基準）。 |
| `silent_mode` | boolean | `false` | 有効にすると上限超過メッセージを静かに破棄。 |

## 仕組み

**セッション**は `{種類}:{ID}` — 特定の個人チャットまたはグループです。EventListener は
`PersonNormalMessageReceived` と `GroupNormalMessageReceived` をフックし、各メッセージで
セッションのカウンターを有効上限（セッション個別の上書き、またはグローバル既定値）と照合します。
上限に達すると、上限メッセージを返信するか静かに破棄（サイレントモード）し、既定の LLM
パイプラインをブロックします。

カウンターは設定タイムゾーンの設定時刻でロールオーバーします。管理ページはプラグインの
`handle_api` を通じて同じメモリ内＋永続化された状態を読み書きするため、変更は即座に反映されます。
